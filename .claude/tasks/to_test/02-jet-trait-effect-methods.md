# Task: Add effect methods to the `Jet` trait

## Summary

Extend the `Jet` trait (or add a supplementary trait) with two methods that expose a jet's side effects: what it reads from the transaction environment, and whether it can cause script failure.

## Details

In `src/jet/mod.rs`, add to the `Jet` trait (or a new `JetEffects` trait):

```rust
/// The set of primitive transaction fields this jet reads from the environment.
/// Returns an empty slice for jets that do not access the environment.
fn read_effects(&self) -> &'static [TransactionField];

/// True if this jet can cause script failure (i.e. it is a verify-style jet).
fn has_write_effect(&self) -> bool;
```

- `read_effects` returns a `&'static [TransactionField]` (a static slice per variant) to avoid allocation.
- `has_write_effect` returns `true` for jets like `CheckSigVerify`, `CheckLockTime`, `CheckLockHeight`, `CheckLockDuration`, `CheckLockDistance`, and any other jet whose C implementation can return `false` (failure).
- Pure computation jets (arithmetic, hashing, etc.) return `false`.
- Jets that read from the environment but do not fail return `false` for `has_write_effect` and a non-empty slice for `read_effects`.
